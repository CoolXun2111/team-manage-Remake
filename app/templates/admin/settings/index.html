System Control

系统设置

按模块集中管理代理、账号安全、日志和自动化任务。

代理配置

控制系统内所有 OpenAI 请求的代理出口，适合海外线路或统一网络转发。

开启后，系统内所有 ChatGPT API 请求都会走代理。

支持 `http://`、`https://`、`socks5://` 和 `socks5h://`。

修改密码

修改管理员密码后，当前会话会跳回登录页重新验证。

建议使用与旧密码不同的组合，避免简单连续字符。

日志级别

控制后台日志的详细程度，排查问题时可临时切到更高详细度。

默认建议使用 `INFO`，只有在排障或接入阶段再切到 `DEBUG`。

补货通知

库存不足时可自动通知外部系统补货，也能同步传递 API Key 供第三方调用。

当系统剩余总席位低于阈值时，会向这里发送补货通知。

剩余总席位小于或等于这个值时触发 Webhook。

用于库存或补货系统对接。

Team 默认限制

控制新导入母号默认可分配的席位上限。

单个 Team 仍可在编辑弹窗里单独调整。

补邀与质保策略

用于处理失效母号后的自动补邀，以及质保期内的重复兑付逻辑。

仅处理拥有质保兑换码的子号邮箱。母号邮箱，尤其是已封禁母号，会自动跳过。

建议不低于 60 秒。时间越短，补邀越快，但 API 请求也会更频繁。

质保规则

质保时长从兑换码首次成功使用后开始计算，后续补邀或重兑不会重置起算时间。

计划设置

按你设定的起始时间和周期自动刷新 Team 状态，适合夜间或固定节奏巡检。

系统会按计划自动同步非封禁 Team 的账号状态、席位和到期信息。

这是首轮执行时间，之后按刷新周期循环运行。

支持 1-24 小时。

执行规则

当前自动刷新任务的实际行为说明，方便你按资源和频率做平衡。

  • 仅同步非封禁 Team，避免对已失效母号做无意义请求。
  • 每轮会刷新账号状态、席位人数和到期信息，沿用现有同步逻辑。
  • 修改时间或周期后，会从新的时间槽重新开始计算执行计划。
  • 如果你设置为 03:00 且周期 6 小时，则会在 03:00、09:00、15:00、21:00 运行。
